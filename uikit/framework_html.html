Buenas prácticas

Nomenclatura

Usaremos la nomenclatura oficial para cada uno de los lenguajes, siempre en inglés:

  • HTML + CSS - Palabras separadas por guiones medios: name-surname

Comentarios*

Cuando sea necesario (solo cuando sea necesario, no abusar) indicar mediante comentarios el incio y el fin de un bloque lo haremos de este modo:

<!-- block-content -->
<div class="block-content">

    ...

</div>
<!-- / block-content -->

Uso correcto de buttons y enlaces

Cuando queramos crear una integración de un modal, un desplegable, un enlace que muestra u oculta algo siempre usaremos etiquetas button, nunca a, los a sólo para enlaces fuera de la página o para hacer anclas dentro del mismo documento, pero nunca para lanzar otros eventos.

Estructura modular

Tal y como comentabamos en el punto anterior, hay que maquetar pensando en los elementos como en módulos u objetos transplantabais que tienen una serie de propiedades que pueden alterarse mediante el uso de clases modificadoras.

Definiremos cada módulo mediante una versión master y luego versiones con diferentes propiedades de este módulo mediante clases modificadoras. Si lo que queremos es alterar las propiedades de todos los módulos de una serie de páginas usaremos una clase modificadora lo más alto posible en el DOM, en el body o en el .wrapper

Todo módulo ha de tener un nombre declarado, los nombres que usemos han de ser lo más abstractos posibles según el tipo de bloque y no según su color o contenido, es mucho mejor un nombre .block-item que un nombre .block-red o .block-video

Master:

<div class="nombre-de-bloque">
    <div class="row">
        <div class="col-sm-6">
            <div class="contenido"></div>
        </div>
    </div>
</div>

Master modificado:

<div class="nombre-de-bloque clase-modificadora">
    <div class="row">
        <div class="col-sm-6">
            <div class="contenido"></div>
        </div>
    </div>
</div>
